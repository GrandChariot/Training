2단원 연습문제

1. CPU의 구성에 대해 설명하시오
- CPU는 연산 장치, 제어 장치, 그리고 레지스터로 이루어진다.
연산 장치는 사칙연산이나 논리연산 등을 맡는 장치이다.
제어 장치는 프로세스나 명령어의 실행 순서 등을 제어하는 장치이다.
마지막으로 레지스터는 CPU가 당장 필요로 하는 데이터를 임시로 보관하는 저장소이다.

2. 폰노이만 구조의 가장 중요한 특징을 설명하시오
- 폰노이만 구조의 알파이자 오메가는 프로그램을 메모리에 올려야 비로소 실행이 가능하다는 사실이다.

3. 버스의 종류를 나열하시오
- 제어 버스, 주소 버스, 데이터 버스

4. 단방향 버스에 대해 설명하시오
- 주소 버스는 오로지 CPU->메모리 방향으로만 사용되는 단방향 버스이다.

5. 다음에 실행할 명령어의 주소를 가지고 있는 레지스터는 무엇인가?
- 프로그램 카운터

6. 다음에 실행할 명령어를 보관하는 레지스터는 무엇인가?
- 명령어 레지스터(*책에는 현재 실행 중인 명령어라고 되어있다)

7. 메모리 주소를 보관하는 레지스터는 무엇인가?
- 주소 레지스터

8. 메모리에 저장할 데이터나 메모리에서 가져온 데이터를 임시로 보관하는 레지스터는 무엇인가?
- 데이터 레지스터(범용 레지스터)

9. 주소 버스와 연결되어있는 레지스터는 무엇인가?
- 메모리 주소 레지스터

10. 데이터 버스와 연결되어있는 레지스터는 무엇인가?
- 메모리 버퍼 레지스터

11. 프로그램을 보호하기 위해 사용하는 두 가지 레지스터는 무엇인가?
- 경계 레지스터와 한계 레지스터

12. 속도 차이가 나는 두 장치 사이에서 속도를 완화하는 장치를 무엇이라고 하는가?
- 버퍼

13. 응용 프로그램과 프린터 사이에서 속도 차이를 완화하는 소프트웨어를 무엇이라고 하는가?
- 스풀러

14. 부팅 시 운영체제를 메모리로 가져오기 위해 사용하는 작은 프로그램은 무엇인가?
- 부트스트랩

15. 속도가 빠르고 값이 비싼 저장장치를 CPU 가까운 쪽에 두고, 그렇지 않은 저장장치를 반대쪽에 배열하는 방식은?
- 저장장치 계층 구조

16. 작업의 효율성을 높이기 위해 CPU가 입출력 관리자에게 입출력 작업을 요청하면 작업을 마친 입출력 관리자는 CPU에게 무엇을 보내는가?
- 인터럽트(하던 작업을 중지하고 입력 데이터를 받아달라)

17. CPU의 도움 없이도 메모리를 사용할 수 있도록 입출력 관리자에게 주는 권한은?
- 직접 메모리 접근(Direct Memory Access)

18. CPU와 입출력 관리자가 서로 다른 메모리 영역을 사용하는 기법을 무엇이라고 하는가?
- 메모리 매핑 입출력

19. CPU가 병렬 처리를 지원하지 않을 때 소프트웨어적으로 병렬 처리를 하는 기법을 무엇이라고 하는가?
- VLIW(Very Long Instruction World) (*잘 안 외워져서 책을 봤다.)

심화 1. 컴퓨터에서 클록이 하는 역할을 설명하시오.
CPU의 모든 구성 요소는 클록이 보내는 신호에 맞춰 움직인다.
이 신호를 클록틱이라고 부르고, 클록틱의 간격이 짧을수록 고성능 CPU이다.

심화 2. 프로그램 카운터 레지스터의 역할을 설명하시오.
다음에 실행할 명령어의 주소를 보관한다.
즉 이 다음에 할 작업이 뭔지 안내하는 역할을 맡고 있다.

심화 3. 제어 버스, 주소 버스, 데이터 버스에 대해 설명하시오.
제어 버스는 다음엔 뭘 할지에 대한 제어 명령이 오가는 버스이다.
주소 버스는 메모리가 저장되는 주소를 주고 받는다.
마지막으로 메모리는 주소 버스로 주고받은 주소에 있는 메모리를 주고 받는다.

심화 4. 휘발성 메모리와 비휘발성 메모리를 비교하여 설명하시오.
휘발성 메모리는 속도가 빠르지만, 전원이 꺼지면 저장된 정보가 사라진다. 그리고 비싸다.
때문에 주로 메인 메모리나 캐시 메모리에 사용된다.
반대로 비휘발성 메모리는 저렴하고 반영구적으로 정보를 저장하지만 느리다.
대표적으로는 SSD나 플래시 메모리, 롬 등이 있다.

심화 5. DDR SDRAM에 대해 설명하시오
기본적으로 DRAM의 D는 Dynamic으로, 가만히 냅두면 정보가 점점 사라진다.
그래서 계속 리프레시 작업을 해줘야 한다.
SDRAM은 클록틱마다 데이터를 다시 활성화하기 때문에 리프레시 작업이 필요없다.
DDR은 Double Data Rate로, 한 번에 2워드만큼씩 데이터를 주고받는다는 의미다.
당연히 속도도 두 배 빠르다.
