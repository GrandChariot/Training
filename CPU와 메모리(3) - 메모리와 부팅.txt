2. 메모리의 종류와 부팅
폰노이만 구조에 의하면 모든 프로그램은 메인 메모리에 들어와야 비로소 실행된다.
그런데, 메모리에도 종류가 정말 다양하다.

- 휘발성 메모리
휘발성 메모리는 전력이 끊기면 데이터도 사라진다.
그럼에도 불구하고 휘발성 메모리가 선호되는 이유는 빠르기 때문!
주로 쓰이는 건 메인 메모리용의 DRAM(Dynamic RAM)과 캐시용의 SRAM(Static)이다(SRAM이 더 빠르다.)
DRAM과 DDR(Double Data Rate)에 대해서는 반도체 전공인 만큼 나름 잘 알고 있다고 자신한다!

- 비휘발성 메모리
플래시 메모리나 SSD, 롬 등은 전원이 끊겨도 데이터가 사라지지 않는 비휘발성 메모리이다.
의외로 USB와 같은 플래시 메모리는 수천 번 이용하면 수명이 다 되니 데이터 보관에 주의하자.


메모리 위주로 돌아가는 현재의 컴퓨터에서 메모리가 강제로 침범당하는 것은 매우 위험하다.
의도적으로 운영체제나 다른 프로그램이 사용하고 있는 메모리 영역을 침범하는 소프트웨어를 '바이러스'라고 한다.

이렇게 메모리 보호를 위해 만들어진 게 '경계 레지스터'와 '한계 레지스터'다.
작업이 시작되면 현재 진행중인 작업의 메모리 시작 주소를 경계 레지스터에 저장하고
이 작업이 사용하는 메모리 크기만큼을 더한 곳의 주소를 한계 레지스터에 저장한다.
다른 작업이나 프로그램이 이 공간을 침범하려고 하거나 작업이 이 밖에서 이루어지면 '인터럽트'가 발생한다.
인터럽트가 발생하면 해당 프로그램은 강제로 종료된다.


그런데 운영체제는 누가 메모리에 올려줄까? 그게 바로 '부팅'이라고 불리우는 작업이다.
전원이 켜지면 자동으로 하드디스크의 1번 섹터(마스터 부트 레코드)에 존재하는 '부트스트랩'을 실행한다.
이 부트스트랩은 운영체제를 메모리에 올려주는 역할을 한다.
부팅 과정을 더 간략하게 살펴보자

전원을 켬 -> 바이오스 실행 -> 마스터 부트 레코드에서 부트스트랩 실행 -> 운영체제 실행
